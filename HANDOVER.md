## Environment gotchas (READ — these will bite a fresh session)
1. **Clone into a dir YOUR sandbox user owns.** A stale `/tmp/work/global-m2` may exist owned by
   `nobody` and be unwritable. Clone fresh to `~/gm` (i.e. `/sessions/<id>/gm`) and
   `git config --global --add safe.directory ~/gm`. The Read/Edit tools CANNOT see `~/gm` or `/tmp`
   — they only reach the mounted outputs dir. Workflow: edit a file in the mounted outputs dir with
   Write/Edit, then `cp` it into `~/gm` for git; read repo files with `cat` in bash.
2. **TradingView IS reachable from the sandbox** (websocket via `tv_pull.pull_series`), so you can
   pull market data and fit/iterate locally — this is how the whole FCI was built. **FRED is NOT
   reachable** from the sandbox (times out); FRED-sourced series (US/Big Picture) can only be
   regenerated by the GitHub Action runner. `web_fetch` is provenance-locked.
3. **`/tmp` can be wiped between bash calls.** Re-fetch defensively. Do file-write + `git add` +
   commit + push in the same bash call when using heredocs.
4. **Pushing** uses an inline token URL, redact it in output:
   `git push "https://x-access-token:<TOKEN>@github.com/Islandboy1968/global-m2.git" HEAD:main`
   then pipe through `sed -E 's/github_pat_[A-Za-z0-9_]+/[REDACTED]/g'`.
5. **Frontend verify is headless** (no browser): strip `<script>` blocks, run the last app script in
   jsdom with a fake `Chart` class that records `data`/`options`/`scales`, bind id'd elements onto
   `window`, eval the app, assert on `window.GLOBAL/US/BIG/CYCLE`. Used all session; reliable.
6. **Trigger + poll the Action** (poll loops time out at ~45s, so poll with single sleeps):
   ```
   curl -fsS -X POST -H "Authorization: Bearer <TOKEN>" -H "Accept: application/vnd.github+json" \
     https://api.github.com/repos/Islandboy1968/global-m2/actions/workflows/main.yml/dispatches -d '{"ref":"main"}'
   # then GET .../actions/runs/<id> until completed/success (~4-6 min). git fetch + rebase after
   # (the Action commits a "data: refresh" on top of your push).
   ```

## GMI FCI reconstruction — methodology (so it's reproducible)
Goal: reconstruct GMI's Financial Conditions Index, which leads ISM ~9–12 months. GMI's own Jan-2026
Monthly: "a regression with an average nine-month lead… optimisers suggest a roughly twelve-month
lead." We use **12mo** as the dashboard default.
- Inputs (TradingView, monthly): blended Treasury rates leg from the 2y/5y/10y
  yields `TVC:US02Y`+`TVC:US05Y`+`TVC:US10Y` (YoY *difference*, pts) — each tenor z-scored, then
  combined dollar-issuance weighted **0.40/0.40/0.20** (2y/5y/10y) and re-z-scored; dollar `TVC:DXY`
  (YoY %), oil `TVC:USOIL` (YoY %). Each leg z-scored to unit variance.
- Composite (looseness) = **−( z(Δrates) + z(Δdollar) + w·z(Δoil) ) / (2+w)**, i.e. rising rates/dollar/oil
  all read as TIGHTER → index falls → ISM falls ~12mo later. `w` = oil weight: **0.5 = headline blend**,
  **0 = ex-oil**. 3-month smoothed, then rescaled to ISM's mean/std so it overlays ISM in PMI units.
- **Copper was tested and dropped** — it's coincident (peaks at 0 lead), so it confirms ISM, it doesn't
  lead it. Oil signed as a *tightening* input (rising oil = worse conditions ahead) is what produces
  the long, strong lead.

## Key analytical findings this session (the macro context)
- **The domino chain (GMI canonical):** Financial Conditions → Liquidity → ISM → Assets, each with a
  lag. Assets are the LAST domino, not simultaneous with liquidity. The FCI is the earliest/faintest.
- **GLI vs ISM lead ≈ 6–7mo; FCI vs ISM lead ≈ 10–12mo** (recent-window cross-correlation). So GLI lags
  FCI by ~3mo and leads ISM by ~6–7mo. Validates the 6mo liquidity / 12mo FCI leads on the dashboard.
- **The 2022-23 FCI overshoot (fell to ~38 while ISM only hit ~46) was a RATES + DOLLAR shock, not
  oil.** Decomposition at the trough: ~−8 rates, ~−6 dollar, <−1 oil; blend and ex-oil bottomed
  identically. The gap to ISM = the capex/stimulus/liquidity cushion absorbing the monetary shock.
- **Ex-oil is the marginally better level-fit recently** (RMSE 4.06 vs 4.38) and the more honest base
  case; the blend's only extra weakness now is the recent oil tick → oil is the swing factor to watch.
- **Liquidity ROC (YoY) is comp-distorted — read the OUTRIGHT LEVEL instead.** Level at all-time highs
  (~136T) but last 3 months nearly flat (+0.35T/mo vs +1T/mo trailing). YoY peaked Jan-2026 (+12.4%),
  now ~8%; this is HARD comps (lapping late-24/early-25). EASY comps (mid-25 plateau) arrive summer/
  autumn 2026 → YoY firms back toward ~9% on continued rises; then HARD comps in Q1-2027 (lapping the
  Dec25-Jan26 surge) collapse it toward 1–3% mechanically. So don't read YoY as a peak signal.
- **GMI's own current house view (Jan/Feb 2026 Monthlies) aligns with "pause not peak":** the liquidity
  short-term top is mechanical (US gov shutdown + TGA rebuild draining liquidity); FC keeps rising at
  the 9–12mo lead "paving the way"; the ISM "went up… with more to come"; and the Jan "Bitcoin: The
  Everything Disconnect" piece argues assets are trading BELOW what liquidity/FCI/regression imply, so
  the setup is catch-up, not a top. Watch-item: if the liquidity level keeps stalling after the
  shutdown/TGA effects clear, AND oil sustains, the pause reclassifies as a peak.
